# JamoMoa v0.9.11

설정에 **개발자 모드**가 새로 추가되었습니다 — 평소엔 숨겨진 옵션이지만,
켜면 변환·복사 동작을 실시간으로 추적할 수 있는 **로그 윈도우**를 메뉴바
와 설정에서 열 수 있습니다. 변환 / 되돌리기 / 실패 / 복사 이벤트를 시간
순으로 보고, 검색으로 필터링하고, 텍스트 파일로 내보낼 수 있습니다. 로그는
앱을 종료해도 다음 실행 시 그대로 복원됩니다.

## 새 기능

- **개발자 모드** — 설정 ▶ 개발자에서 활성화. 메뉴바에 *로그 표시* 항목이
  나타나고, 별도 윈도우로 모든 파일명 변환 / 되돌리기 / 복사 이벤트를
  타임스탬프와 함께 표시. 분해된(NFD) 이름은 *자모를 풀어쓴 형태*로 보여
  바이트 단위 차이가 시각적으로 드러납니다 (예: `[ㅈㅔ9ㅎㅗㅣ.jpg] →
  [제9회.jpg]`).
- **로그 검색** — 윈도우 상단 검색 필드로 메시지 내용 즉시 필터링.
- **로그 저장** — 화면에 보이는 (필터된) 엔트리를 텍스트 파일로 export
  (`로그 저장…` 버튼).
- **로그 영속화** — 앱을 종료해도 로그가 사라지지 않고 다음 실행 시 그대로
  복원.
- **윈도우 상태 복원** — 메인 윈도우와 로그 윈도우가 종료 직전 열려 있던
  상태로 재실행 시 같이 떠올라옴.

## 변경

- **이름 복사 시 자모 합치기 OFF의 의미 명확화** — 이제 OFF는 *변환 안 함*
  (디스크에 있는 그대로)을 의미합니다. 이전엔 OFF가 *NFD로 강제 분해*여서
  이미 NFC 상태인 파일을 복사할 때 의도치 않게 자모가 분리된 형태로 클립
  보드에 들어갔습니다. 이제 NFC 파일은 NFC 그대로, NFD 파일은 NFD 그대로
  복사됩니다.
